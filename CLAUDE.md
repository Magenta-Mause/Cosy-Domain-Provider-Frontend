# Cosy Domain Provider ŌĆö Frontend

React SPA for managing `cosy-hosting.net` subdomains. Backend: `../cosy-domain-provider` (Spring Boot).

## Commands

```bash
npm run dev        # Dev server at localhost:5173 (proxies /api ŌåÆ http://127.0.0.1:8080)
npm run build      # TypeScript check + Vite production build
npm run lint       # Biome check
npm run lint:fix   # Biome auto-fix
npm run gen:api    # Regenerate API client from running backend OpenAPI spec (backend must be up)
```

## Key conventions

- **Styling: Tailwind utility classes first.** Prefer `className="..."` over `style={{...}}`. Only use inline styles for values that cannot be expressed as Tailwind classes (oklch/gradient/textShadow/boxShadow literals, dynamic numeric values with no Tailwind equivalent, or CSS properties with no utility class). See `docs/patterns/styling.md`.
- **Tailwind design tokens:** The design system's CSS variables are registered as Tailwind tokens. Use named utilities like `bg-background`, `text-foreground`, `border-destructive`, `rounded-radius-sm` ŌĆö never the raw `[var(--x)]` arbitrary syntax.
- **File naming:** Component files use **kebab-case** (e.g. `user-menu.tsx`, `billing-header.tsx`). Logic hook files use **camelCase** (e.g. `useDashboardLogic.ts`, `useDropdown.ts`).
- **Linting/formatting: Biome only.** No ESLint, no Prettier. Run `npm run lint:fix` before committing.
- **Never edit generated files:** `src/routeTree.gen.ts` (auto-generated by TanStack Router on dev start) and `src/api/generated/` (generated by Orval from OpenAPI spec).
- **i18n:** Translation keys live in `src/i18n/resources.ts` as TypeScript (not JSON). Always add keys to **both** `en` and `de`.
- **API client:** Custom logic goes in `src/hooks/useDataLoading` and `src/hooks/useDataInteractions`, not in the generated API code.
- **New routes:** Create the page in `src/pages/`, create the route file in `src/routes/`. The route tree regenerates automatically on next dev server start.
- **Testing:** Use `data-testid` attributes for Playwright selectors (`page.getByTestId(...)`).
